Características — Web to PDF / Guardar como PDF

本页面为站内工具页模板，详细说明了“网页转 PDF”的功能、本地实现方式和适用场景，便于用户快速理解并信任工具。页面内容兼顾技术细节与使用指南，满足 Google EEAT（专业性/经验/权威性/可信度）。

  • 高保真打印：通过浏览器的打印（`window.print()`）生成 PDF，保留页面 CSS、字体与排版，适合生成可打印、分页友好的 PDF 文件。
  • 可搜索文本：优先使用浏览器原生打印保证文本可选中、可复制，保留可访问性语义（屏幕阅读器友好）。
  • 截图备选方案：当打印无法满足（例如复杂 canvas 或动态内容）时，页面会加载 `html2canvas` 并导出高分辨率 PNG 作为备选。
  • 隐私保护：所有操作均在用户浏览器本地完成，不上传任何页面内容到第三方服务器（详见下方“关于和隐私”）。
  • 跨站点兼容：使用 `useCORS` 选项和动态脚本加载以提高兼容性，但受限于目标页面的 CSP / 广告拦截器。

还可以试用：网页转图片 (Web to Image) — 将整个页面导出为 PNG 图像。

Acerca de esta herramienta (Cómo funciona & EEAT)

作者与维护：本工具由 TableCopy 团队和开源社区协作维护，源代码及实现位于本项目的 tools/tablecopy/ 目录，欢迎审计与贡献。

实现原理：首选调用浏览器的打印子系统（`window.print()`）以保留页面文本和样式；若需图像备份则动态加载 `html2canvas`（CDN）进行截图并下载 PNG。代码在页面中以书签（bookmarklet）方式注入，相关 JS 逻辑示例位于 web-converter.js。

可验证的安全性与隐私：不含远程数据上报；截图与 PDF 在客户端生成并直接下载。第三方库（如 html2canvas）从公开 CDN 加载，脚本在加载失败时提供友好提示。

最佳实践（建议）：在打印前关闭浮动工具栏与模态窗口；对长页建议先使用“页面设置”调整纸张与缩放以获得最佳分页效果。

Preguntas Frecuentes (FAQ)

¿Esta herramienta subirá el contenido de mi página?

No. Todo el procesamiento (impresión o captura) se realiza localmente en el navegador y el archivo exportado se descarga en su dispositivo.

¿Por qué el PDF impreso difiere de la página en pantalla?

Muchos sitios incluyen estilos @media print que cambian la salida de impresión. Verifique la vista previa de impresión para paginación, escala y márgenes; use el modo de captura para fidelidad visual.

¿Cuándo debo usar captura (PNG) en lugar de imprimir?

Cuando la página contiene canvas complejos, WebGL o visualizaciones dinámicas, imprimir puede no renderizar correctamente; use el modo captura para capturar la salida visual.

¿Cómo preservar texto buscable al imprimir?

Prefiera la impresión nativa del navegador (window.print()) que preserva el texto del DOM. Evite el modo captura si necesita texto seleccionable/buscable.

¿Y si las bibliotecas CDN fallan por bloqueadores?

Si html2canvas u otras bibliotecas CDN fallan, el script muestra un mensaje de fallback y sugiere usar la impresión del navegador o permitir el CDN.

¿Cómo reporto bugs o contribuyo?

Abra un issue o PR en el repositorio (ruta: tools/tablecopy/) con un ejemplo reproducible o grabación.
